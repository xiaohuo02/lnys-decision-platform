# 前端架构决策（v4.0）

## 1. 结论

当前项目前端 **继续沿用 `Vue 3 + Vite + Element Plus + ECharts`**，不在本阶段切换到 Next.js、React 或 shadcn/ui。

v4.0 的治理后台与业务前端统一放在现有 `frontend/` 工程中增量实现，通过路由、页面分组和组件规范完成隔离。

---

## 2. 为什么这样决定

### 2.1 保留现有技术栈的原因

- 当前仓库已经存在可运行的 `Vue 3 + Vite` 前端基础
- 团队现阶段的主要目标是把 v4.0 架构落地，而不是重做前端框架迁移
- 治理后台本质是高密度信息管理界面，`Element Plus + ECharts` 足够支撑
- 保持现有技术栈可以减少与后端联调成本，更符合 `main/dev/feature/*` Git 节奏

### 2.2 暂不切换 Next.js 的原因

- Next.js 更适合从零搭建复杂控制台，但当前项目已有 Vue 资产
- 直接换栈会让任务从“实现治理后台”变成“重建整个前端”
- 当前评审与答辩更需要可运行结果，而不是重新选型带来的形式提升

## 3. 页面组织方式

```text

├── views/
│   ├── business/
│   │   ├── dashboard/

│   │   ├── inventory/
│   │   ├── reports/
│   └── console/
│       ├── dashboard/
│       ├── workflows/
│       ├── agents/

│       ├── evals/
│       ├── reviews/
```

## 3.2 页面布局原则

- 业务前端：偏业务图表、表单和报告浏览
- 治理后台：偏高密度表格、Trace 树、时间线、Diff 视图
- 详情页优先采用双栏或三栏布局
- 弹窗只用于轻操作，复杂编辑尽量使用独立页面或右侧抽屉

---

## 4. UI 风格约束

v4.0 的 UI 风格定义为：**Twitter + Codex 风格的扁平化治理后台**。

### 4.1 具体落地要求

- 信息密度高，优先列表/表格/时间线
- 颜色克制，深色模式优先，浅色模式兼容
- 状态用少量颜色点缀，不依赖大面积渐变
- 关键 ID、版本号、延迟、token 等字段使用等宽字体展示
- Trace、JSON、Diff 页面强调可读性与对齐

### 4.2 与 Element Plus 的适配方式

- 基础表单、表格、抽屉、Tabs 沿用 Element Plus
- 图表统一使用 ECharts
- 对治理后台页面统一封装一层 `console-*` 样式体系
- 通过主题变量控制深色/浅色两套视觉 token

---

## 5. 前端状态与数据策略

### 5.1 当前阶段

- 继续使用 `Pinia` 作为轻量状态管理
- 页面数据通过 `axios` 访问后端 API
- 不引入额外前端服务层框架

### 5.2 数据边界

- 页面只保留展示需要的视图状态
- 工作流状态、trace 数据、评测结果以服务端为准
- 大对象内容（如 artifacts、trace payload）按需加载，不一次性全量拉取

---

## 6. Console 的最小实现范围

第一阶段必须具备以下页面：

- Dashboard
- Workflows
- Agents
- Traces & Logs
- HITL 审核
- Prompt Center
- Policies & Guardrails
- Release Center（至少包含列表页与回滚详情页）
- Audit
- Team（用户管理与角色分配）

第二阶段补齐：

- Evals
- Knowledge Center
- Memory Center
- Release Center（高级发布编排与版本对比）
- Settings

---

## 7. Git 协作约束

前端任务必须按页面或能力拆分，不允许一个分支同时改动大量无关页面。

推荐分支命名：

- `feature/console-dashboard`
- `feature/console-workflows`
- `feature/console-traces`
- `feature/console-prompts`
- `feature/business-report-view`

每个 PR 至少说明：

- 新增页面或组件
- 新增 API 依赖
- 是否影响现有业务页面
- 是否需要联调数据或 mock 数据

---

## 8. 暂缓事项

以下内容不在当前阶段执行：

- 全量迁移 React / Next.js
- 组件库整体替换为 shadcn/ui
- 前端改造成多应用 monorepo
- 引入复杂 SSR/ISR 能力

当前前端架构的核心目标是：**在最小迁移成本下支撑 v4.0 治理后台和业务分析场景。**
