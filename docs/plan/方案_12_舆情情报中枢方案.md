# 舆情情报中枢（Sentiment Intelligence Hub）— 开发方案 v1.0

> 最后更新: 2026-04-07
> 负责人: 舆情分析模块开发者
> 状态: 待实施

---

## 一、目标

将当前"单次查询式"舆情分析升级为 **持续积累的情报系统**：

1. **ASTE 四元组提取** — 从每条评论动态提取 `(实体, 方面, 观点词, 情感)` 结构化信号
2. **向量知识库沉淀** — 分析结果 embedding 后写入 ChromaDB，支持语义检索
3. **跨智能体信号分发** — 通过 Redis Pub/Sub 实时推送 + RAG 按需拉取
4. **HITL 反馈闭环** — 人工审核结果回流知识库，持续提升系统

**核心展示价值**: 多智能体通过共享 embedding 空间协同决策，而非各自独立运行。

## 二、现有架构（已完成）

```
用户输入文本
     │
     ▼
┌─ sentiment_service.py ──────────────────────────┐
│  Step 1: BERT-Chinese 本地推断                    │
│  Step 2: cascade_analyze() (sentiment_llm_service)│
│    Tier 1: BERT conf ≥ 0.92 → 直出               │
│    Tier 2: LLM CoT 单次推理                       │
│    Tier 3: LLM Self-Consistency 三路投票           │
│    Tier 4: 标记 uncertain → HITL                  │
│  Step 3: 结果写入 Redis 缓存                      │
└─────────────────────────────────────────────────┘
```

**当前输出结构** (`SentimentAnalyzeResponse`):

**问题**: aspects 固定 4 维度，无实体提取，无跨 Agent 信号，无知识沉淀。

#### 6.1.2 core/embedding.py

```python
"""bge-small-zh-v1.5 单例服务 — 全局共享，仅 200MB"""

class EmbeddingService:
    _instance = None

    def __init__(self):
        from sentence_transformers import SentenceTransformer
        from backend.config import settings
        self.model = SentenceTransformer(settings.BGE_MODEL_NAME)
        self.dim = self.model.get_sentence_embedding_dimension()

    @classmethod
    def get_instance(cls) -> "EmbeddingService":
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance

    def embed(self, texts: list[str]) -> list[list[float]]:
        return self.model.encode(
            texts, normalize_embeddings=True, show_progress_bar=False
        ).tolist()

    def embed_one(self, text: str) -> list[float]:
        return self.embed([text])[0]
```

**资源预算**: 33M 参数，~200MB 内存，~15ms/句 CPU 推理。

### Phase 2: LLM Prompt 升级为 ASTE 提取 (P0, ~1h)

#### 6.2.1 sentiment_llm_service.py — _SYSTEM_PROMPT 替换

**现有 prompt 输出**:
```json
{
  "aspects": {"质量":"正面","物流":"负面","服务":"无关","价格":"无关"}
}
```

**升级后 prompt 输出**:
```json
{
  "label": "负面",
  "confidence": 0.85,
  "reasoning": [...],
  "key_phrases": ["牛轧糖不错","大黄鱼不好吃"],
  "entity_sentiments": [
    {"entity":"牛轧糖","aspect":"口味","opinion":"不错","sentiment":"正面"},
    {"entity":"大黄鱼","aspect":"口味","opinion":"不好吃","sentiment":"负面"}
  ],
  "intent_tags": ["quality_complaint","cross_product_compare"],
  "agent_signals": [
    {
      "target_agent": "inventory",
      "signal_type": "quality_alert",
      "entity": "大黄鱼",
      "severity": "medium",
      "suggestion": "口味差评集中，建议检查供应批次"
    }
  ]
}
```

**Prompt 设计要点**:
- 保留原有 4 步推理链（关键词识别 → 否定词分析 → 隐式情感 → 综合判定）
- 新增第 5 步"实体方面提取"，要求动态识别所有提及的实体+方面+观点
- 新增第 6 步"跨智能体信号"，要求输出 target_agent + signal_type
- entity_sentiments 无上限，有多少提取多少
- intent_tags 从预定义枚举中选择
- agent_signals 只在有明确业务价值时输出（不强制）

**向后兼容**:
- 保留 `aspects` 字段（从 entity_sentiments 聚合生成），前端旧逻辑不受影响
- 新字段均为 Optional，cascade_analyze() 返回结果兼容不含新字段的情况

#### 6.3.2 sentiment_kb_service.py

```python
class SentimentKBService:
    """舆情情报知识库 — 写入/语义检索/跨Agent查询"""

    def __init__(self):
        self.embedding = EmbeddingService.get_instance()
        self.store = VectorStoreManager.get_instance()
        self.reviews = self.store.get_collection("sentiment_reviews")

    async def ingest(self, review_id: str, text: str, result: dict) -> str:
        """分析完成后写入知识库"""
        vec = self.embedding.embed_one(text)
        self.reviews.upsert(
            ids=[review_id],
            embeddings=[vec],
            documents=[text],
            metadatas=[{
                "label": result.get("label", ""),
                "confidence": result.get("confidence", 0),
                "model_used": result.get("model_used", ""),
                "entity_sentiments": json.dumps(
                    result.get("entity_sentiments", []), ensure_ascii=False
                ),
                "intent_tags": json.dumps(
                    result.get("intent_tags", [])
                ),
                "agent_signals": json.dumps(
                    result.get("agent_signals", []), ensure_ascii=False
                ),
                "cascade_tier": result.get("cascade_tier", 0),
                "timestamp": datetime.now().isoformat(),
            }]
        )
        return review_id

    async def search_similar(self, query: str, top_k=5, where=None):
        """语义检索 — 其他 Agent 调用入口"""
        vec = self.embedding.embed_one(query)
        results = self.reviews.query(
            query_embeddings=[vec],
            n_results=top_k,
            where=where,
        )
        return results

    async def search_by_entity(self, entity: str, days=7, top_k=20):
        """按实体检索近 N 天评论 — 库存/客户 Agent 调用"""
        vec = self.embedding.embed_one(entity)
        cutoff = (datetime.now() - timedelta(days=days)).isoformat()
        return self.reviews.query(
            query_embeddings=[vec],
            n_results=top_k,
            where={"timestamp": {"$gte": cutoff}},
        )

    async def get_stats(self):
        """知识库统计信息"""
        return {
            "total_reviews": self.reviews.count(),
            "collection": "sentiment_reviews",
        }
```

#### 6.3.3 sentiment_service.py — analyze() 扩展

在现有 `analyze()` 方法的 `return ok(result)` **之前**，追加:

```python
# ★ Step 4: 写入知识库 + 信号分发
if settings.SENTIMENT_KB_ENABLED and cascade_result is not None:
    try:
        from backend.services.sentiment_kb_service import SentimentKBService
        kb_svc = SentimentKBService()
        kb_id = await kb_svc.ingest(cache_key, body.text, result)
        result["kb_id"] = kb_id

        # ★ Step 5: Redis Pub/Sub 分发高优信号
        signals = result.get("agent_signals", [])
        for sig in signals:
            if sig.get("severity") in ("medium", "high"):
                await self.redis.publish(
                    f"sentiment:signal:{sig['target_agent']}",
                    json.dumps(sig, ensure_ascii=False)
                )

        # ★ Step 6: 实体级累积计数
        for es in result.get("entity_sentiments", []):
            if es.get("sentiment") == "负面":
                counter_key = f"sentiment:neg_count:{es['entity']}:7d"
                count = await self.redis.incr(counter_key)
                await self.redis.expire(counter_key, 7 * 86400)
    except Exception as e:
        logger.warning(f"[sentiment_svc] KB ingest/signal error: {e}")
```

## 十一、风险与回退

| 风险 | 概率 | 应对 |
|---|---|---|
| bge-small-zh 下载慢/失败 | 中 | 提前下载到 models/ 目录，离线加载 |
| ChromaDB 写入性能不足 | 低 | 异步写入，不阻塞主请求 |
| LLM 新 prompt 输出格式不稳定 | 中 | JSON 解析时 fallback 到旧格式 |
| 内存超限 | 低 | bge-small 仅 200MB，总预算充足 |
| 队友的 KB 和你的冲突 | 低 | collection 级别完全隔离 |

**回退策略**: `SENTIMENT_KB_ENABLED=False` 一键关闭所有新功能，回到纯 Cascade 模式。
