## 一、数据流通用模式

后端每个 GET 接口的数据查找顺序（三级降级）：

```
1. Redis 缓存命中 → 直接返回 ok(data)
2. CSV 文件命中   → 读取 → 回写 Redis + DB → 返回 ok(data)
3. Mock 降级      → ENABLE_MOCK_DATA=true 时返回 degraded(mock_data)
4. 全部失败       → 返回 error("暂未就绪", code=503)
```

统一响应结构（2026-04-03 更新）：

```json
{
  "code": 200,
  "data": { ... },
  "message": "ok",
  "meta": {
    "degraded": false,
    "source": "csv",
    "trace_id": null,
    "warnings": []
  }
}
```

前端拦截器自动将 `meta` 附加为 `data._meta`（不可枚举属性），业务组件通过 `d?._meta?.degraded` 检测降级状态。

> **2026-04-05 更新**：`requestBusiness` 和 `requestAdmin` 拦截器现均支持 `_meta` 附加。

POST 接口的数据查找顺序：

```
1. Redis 缓存命中 → 返回 cached(data)
2. Agent 调用     → agent 就绪则调用 agent.run() 返回结果
3. Agent 不可用   → 走 Service 降级层（ML模型/规则引擎/关键词匹配）
4. 全部失败       → 返回 mock 结果
```

## 4.1 ID 语义说明

| ID 字段 | 含义 | 生成方 | 生命周期 | 跨接口传递 |
|---------|------|--------|---------|-----------|
| `run_id` | 单次 workflow 执行的唯一标识 | 后端 `/analyze` 或 `/workflows/run` 生成 (UUID) | 从启动到完成/失败/取消 | `status` / `cancel` / `stream` / `traces` 均使用 |
| `thread_id` | LangGraph checkpoint 线程标识，同一对话/会话可复用 | 前端可指定，否则后端生成 (UUID) | 可跨多次 run 复用 | `/analyze` 请求体传入，checkpoint 持久化使用 |
| `session_id` | OpenClaw 客服会话标识 | 前端 `useChatStore.newSession()` 生成 | 单次客服对话会话 | `/chat/message` + `/chat/history/{session_id}` |
| `case_id` | HITL 审核用例标识 | 后端欺诈风控流程创建 | 审核完成前有效 | `/reviews/{case_id}/*` |
| `artifact_id` | workflow 产出物标识 | 后端 workflow 执行时写入 | 永久 | `/artifacts/{artifact_id}` |

> **注意**：`run_id` 和 `thread_id` 虽然都是 UUID，但语义不同。`run_id` 标识一次执行，`thread_id` 标识一个 checkpoint 线程（可包含多次执行）。前端不应混用。

## 4.2 SSE 事件契约（`/api/v1/workflows/{run_id}/stream`）

| 事件类型 | 触发时机 | data 字段 | 是否终止流 |
|---------|---------|----------|-----------|
| `route_decided` | SupervisorAgent 路由完成 | `{workflow, modules, confidence}` | 否 |
| `step_started` | workflow 节点开始执行 | `{step_name, message}` | 否 |
| `step_completed` | 节点执行完成 | `{step_name, message, latency_ms, progress_pct}` | 否 |
| `step_failed` | 节点执行失败 | `{step_name, message}` | 否 |
| `hitl_required` | 需要人工审核（欺诈高风险） | `{step_name, case_id, message}` | 否（流暂停等待） |
| `hitl_resolved` | 人工审核完成 | `{step_name, decision, message}` | 否 |
| `final` | workflow 全部完成 | `{status, run_id, executive_summary, ...}` | **是** |
| `error` | workflow 失败 | `{run_id, error, message}` | **是** |

> - 前端收到 `final` 或 `error` 后应关闭 EventSource 连接
> - SSE 不支持 replay，新订阅者只能收到订阅之后的事件
> - 心跳间隔 15s，最大流持续 600s
