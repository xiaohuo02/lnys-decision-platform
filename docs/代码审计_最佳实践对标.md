## 二、P0 — 必须修复

### P0-1: `datetime.utcnow()` 已弃用 (49处/21文件)
- **问题**: Python 3.12 正式弃用 `datetime.utcnow()`，3.14 将移除
- **影响**: 运行时 DeprecationWarning，未来版本不兼容
- **修复**: `datetime.utcnow()` → `datetime.now(timezone.utc)`
- **涉及**: schemas/, agents/workflows/, services/, governance/, middleware/

### P0-2: Pydantic v2 残留 v1 API
- **问题 A**: 6处 `class Config:` 应迁移为 `model_config = ConfigDict(...)`
  - `ErrorAction`, `SupervisorOutput`, `RunRecord`, `StepRecord`, `ServiceResult` 及多个 schema
- **问题 B**: 15处 `.dict()` 应迁移为 `.model_dump()`
  - `business_overview.py` (6处), `risk_review.py` (1处), `tracer.py` (1处), `internal/services.py` (7处)
- **问题 C**: `use_enum_values = True` 应迁移到 `model_config = ConfigDict(use_enum_values=True)`
- **影响**: Pydantic v2 兼容模式运行，性能未充分利用，未来版本将移除

### ~~P0-3: `app_error_handler` 始终返回 HTTP 200~~ ✅ 已修复 (2026-04-04)
- **问题**: `exceptions.py` 的 `app_error_handler` 返回 `status_code=200`，客户端无法通过 HTTP 状态码判断错误
- **修复**: 已改为 `http_status = exc.code if 400 <= exc.code < 600 else 500`，正确返回 HTTP 状态码

### ~~P0-4: SECRET_KEY 生产环境缺少校验~~ ✅ 已修复 (2026-04-03)
- **问题**: `config.py` 默认值 `"change_this_in_production"`，生产部署可能遗忘修改
- **修复**: `model_validator` 中校验生产环境 SECRET_KEY 长度 ≥ 32 字符，DEV_BACKDOOR 禁止开启

### P0-5: JWT `datetime.utcnow()` 安全问题
- **问题**: `middleware/auth.py` 使用 `datetime.utcnow()` 生成 JWT 过期时间
- **修复**: 使用 `datetime.now(timezone.utc)` 确保时区感知

## 四、P2 — 建议优化

- 整个前端为纯 JavaScript，建议渐进式迁移到 TypeScript
